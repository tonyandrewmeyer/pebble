# Contributing to Pebble

Thank you for your interest in contributing to Pebble!

Developer documentation lives in [HACKING.md](HACKING.md): how to build and run
the daemon, use the CLI client, hit the API with curl, follow the code style,
run the tests, build the docs, and create a release.

Changes are proposed as
[pull requests on GitHub](https://github.com/canonical/pebble/pulls). Please
format your commit and pull-request titles following the
[conventional commit](https://www.conventionalcommits.org/en/v1.0.0/) style, as
described in [HACKING.md](HACKING.md#commits).

This project follows the
[Ubuntu Code of Conduct](https://ubuntu.com/community/ethos/code-of-conduct).
